## 真实 AI 端到端验收

| 步骤 | 操作 | 通过标准 | 状态 |
|---:|---|---|---|
| 1 | 准备一段不少于 20 字的 JD | 可提交给后端 | 通过 |
| 2 | 准备一份 `txt/docx/pdf` 文本简历 | 后端能解析出文本 | 通过，使用 `sample-resume.txt` |
| 3 | 创建 Agent 任务 | `/api/agent/jobs` 返回 `202` 与 `jobId` | 通过，返回 `202` |
| 4 | 轮询 Agent 任务 | 6 个阶段依次进入 `completed` | 通过 |
| 5 | 查看匹配报告 | 报告包含候选人、岗位、分数、建议、风险 | 通过，结果保存至 `output/last-agent-job.json` |
| 6 | 观察失败场景 | 模型失败/超时时返回清晰错误，不泄漏密钥 | 通过，自动化测试覆盖 400/415/502、断连与 AbortError 映射；真实超时场景未手动触发 |

## 异常场景验收

| 场景 | 操作 | 预期 | 状态 |
|---|---|---|---|
| 后端未启动 | 前端触发分析 | 提示无法连接后端服务 | 通过，前端测试覆盖 |
| JD 内容过短 | 提交短文本 | 返回校验错误 | 通过，前端本地校验覆盖 |
| 不支持的简历类型 | 上传非 `txt/docx/pdf` | 返回不支持文件类型 | 通过，前端本地校验与后端 415 覆盖 |
| 空简历文本 | 上传空文本文件 | 返回无可用文本提示 | 通过，前端 TXT 空文件校验覆盖 |
| AI 服务失败或超时 | 模型返回失败或请求中止 | 返回可理解提示，不白屏、不卡死 | 通过，前端 502 与 AbortError 映射覆盖 |
| Agent 运行失败 | Agent 任一阶段失败 | Dashboard 显示失败阶段和失败原因 | 通过，前端测试覆盖 |
| 模型配置缺失 | 移除本地 API Key 后调用 | 返回 AI 配置/服务错误 | 自动化覆盖服务错误映射；真实环境不建议现场移除密钥 |
