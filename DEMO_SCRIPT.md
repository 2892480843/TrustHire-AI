# 比赛演示脚本

本脚本用于本地比赛 Demo 演示。演示时不要打开、朗读、截图或分享 `.env` 中的真实密钥。

## 演示目标

| 目标 | 讲清楚什么 |
|---|---|
| 可信招聘闭环 | 系统从 JD、简历、证据链、评分、面试任务到报告形成完整决策链。 |
| AI Agent 编排能力 | Dashboard 可一键串联 6 个分析阶段，并展示每个阶段状态。 |
| HR 可解释决策 | 每个推荐结论都能回到岗位要求、简历证据和风险点。 |
| Demo 稳定性 | 正常链路能跑通，异常场景有清晰提示，不白屏、不卡死。 |

## 启动命令

首次运行或依赖变化时：

```powershell
npm install
```

本地准备环境变量文件：

```powershell
Copy-Item .env.example .env
```

只在本机 `.env` 中填写真实配置，不要把真实密钥写入文档、截图、测试或日志。

启动前后端：

```powershell
npm run dev:full
```

演示入口：

| 服务 | 地址 |
|---|---|
| 前端 | `http://localhost:5173` |
| 后端健康检查 | `http://localhost:3001/health` |

## 标准演示路线

| 顺序 | 页面 | 操作 | 讲解重点 |
|---:|---|---|---|
| 1 | Dashboard | 打开首页 | 这是 HR 的招聘分析工作台，展示流程进度、候选人状态、风险数量和 Agent 入口。 |
| 2 | JD 解析 | 点击 `JD 解析`，使用默认 JD 或粘贴比赛岗位 JD，点击解析 | 系统把自然语言 JD 转成能力模型、权重和筛选建议。 |
| 3 | 简历分析 | 点击 `简历分析`，上传 `sample-resume.txt`；如需兜底，可在 Dashboard 点击 `使用样例数据（演示）` 注入完整样例链路 | 系统提取候选人基础信息、技能、项目经历和风险点。 |
| 4 | 证据链分析 | 进入 `证据链分析` | 讲每项能力要求如何被简历证据支持，哪些地方证据不足。 |
| 5 | 匹配评分 | 进入 `匹配评分` | 展示总分、维度分、优势、短板和 HR 建议。 |
| 6 | 面试任务 | 进入 `面试任务` | 展示 AI 生成的技术追问、项目核验、实操任务和风险追问。 |
| 7 | 匹配报告 | 进入 `匹配报告` | 展示最终建议、风险、缺口、面试建议，并演示导出 Markdown。 |

## Agent 演示步骤

使用脱敏样例简历 `sample-resume.txt`。

1. 确认后端健康检查可访问：`http://localhost:3001/health`。
2. 打开 `http://localhost:5173`，停留在 Dashboard。
3. 在 `Agent JD 输入` 中保留默认 JD，或粘贴一段不少于 20 个字符的岗位 JD。
4. 点击 `选择简历`，选择项目根目录下的 `sample-resume.txt`。
5. 点击 `一键 Agent 分析`。
6. 观察 6 个阶段依次从 `等待`、`运行中` 到 `已完成`：
   - JD 解析
   - 简历解析
   - 证据链分析
   - 人岗匹配评分
   - 面试任务生成
   - 匹配报告生成
7. 任务完成后，说明系统已把完整结果写入前端流程状态，可继续进入报告页查看。

如现场网络或模型响应较慢，可在 Dashboard 点击 `使用样例数据（演示）` 保持讲解节奏。该按钮会直接注入完整 JD、简历、证据链、评分、面试任务和匹配报告；随后进入 `匹配报告` 页即可展示 `张同学` 的完整报告，并演示导出 Markdown。真实端到端结果仍可引用 `output/last-agent-job.json` 作为赛前验收证据。

## 页面讲解词

| 页面 | 建议讲法 |
|---|---|
| Dashboard | “这里不是单点聊天机器人，而是招聘流程工作台。HR 能看到流程进度、风险数量，也能一键启动 Agent，把后续页面全部串起来。” |
| JD 解析 | “第一步把 JD 拆成能力模型。权重用于后续评分，筛选建议用于 HR 初筛口径统一。” |
| 简历分析 | “系统只使用脱敏样例简历演示。它会把非结构化文本转成候选人信息、技能、项目和风险点。” |
| 证据链分析 | “可信招聘的关键是证据。这里展示岗位能力和简历证据之间的对应关系，以及缺证据项。” |
| 匹配评分 | “评分不是黑盒总分，而是按技能、项目、潜力、风险等维度拆开，便于 HR 复核。” |
| 面试任务 | “面试问题来自前面的证据链和风险项，能帮助面试官验证真实性，而不是泛泛提问。” |
| 匹配报告 | “最终报告把分数、优势、风险、缺口和建议汇总，便于导出、复盘和团队协作。” |

## 现场异常兜底话术

| 异常 | 页面提示 | 兜底话术 |
|---|---|---|
| 后端未启动 | `无法连接后端服务，请确认 server 已启动。` | “这是本地服务未启动，不是页面崩溃。启动 `npm run dev:full` 后即可恢复。” |
| JD 内容过短 | `JD 内容至少需要 20 个字符。` | “系统会先做基础校验，避免把无效输入送进模型浪费时间。” |
| 不支持的简历格式 | `仅支持 PDF / DOCX / TXT 简历文件。` | “当前 Demo 只开放常见文本简历格式，避免图片等无法解析内容影响判断。” |
| 空简历文本 | `简历文件为空，请上传包含文本内容的 PDF / DOCX / TXT 简历。` | “系统会阻断空文件；扫描版 PDF/OCR 是生产化扩展项。” |
| AI 服务失败或超时 | `AI 服务调用失败，请稍后重试。` 或 `AI 服务响应超时，请稍后重试，或先使用样例数据完成演示。` | “外部模型服务异常不会导致白屏，Demo 可切换完整样例数据继续展示业务闭环和报告导出。” |
| Agent 运行失败 | Dashboard 显示失败阶段和失败原因 | “Agent 的每一步都有状态记录，失败时能定位到具体阶段，便于重试或排查。” |
